### Task 1: Go models + handler 响应/Usage + Redis key 对齐 Python（point 1-3）

**Files:**
- Modify: `services/go/quota/internal/models/types.go`（QuotaCheckResponse / UsageResponse / UsagePoint）
- Modify: `services/go/quota/internal/handler/quota.go`（usage 返回扁平；check 填 retry_after_seconds）
- Modify: `services/go/quota/internal/limiter/redis.go`（rateKey 格式 + CheckAndConsume 填 retry_after_seconds）

**对齐基准（Python `models.py`）：**
```python
QuotaCheckResponse{allowed, tier_blocked, limit, remaining, retry_after_seconds, rule_source}
UsagePoint{window_seconds, used, limit}   # limit 可 None
UsageResponse{tenant_id, app_id, api_id, second, minute, day}  # 扁平
```

- [ ] **Step 1: models 对齐** — `QuotaCheckResponse` 去 `Current`/`ResetMs`，加 `RetryAfterSeconds int json:"retry_after_seconds"`；`UsagePoint{WindowSeconds,Used,Limit *int}`（Limit 指针=可 null）；`UsageResponse{TenantID,AppID,APIID string; Second,Minute,Day UsagePoint}`（扁平，去 Points 数组）。
- [ ] **Step 2: limiter rateKey + retry_after** — `rateKey(tenant,api,app,tier,slot)` → `fmt.Sprintf("t:%s:rate:%s:%s:%s:%s", tenant, api, app, tier /*"s"|"m"|"d"*/, slot)`（去 region 前缀，加 tier）；CheckAndConsume 被挡时 `RetryAfterSeconds = 窗口剩余秒`（TTL 或 windowMs 推算），allowed=True 时 0。
- [ ] **Step 3: handler usage 扁平** — `usage` 构造 `UsageResponse{TenantID,AppID,APIID,Second,Minute,Day}`（limiter.GetUsage 返回扁平 3 point）。
- [ ] **Step 4: 测试** — 对照 Python models 逐字段断言（JSON marshal 形状一致）；rateKey 断言 `t:tenant_a:rate:api_x:app_y:s:<slot>`。
- [ ] **Step 5: go test + vet + commit**

### Task 2: limiter Lua 原子（point 4，WIP redis.go 输入）

**Files:**
- Modify: `services/go/quota/internal/limiter/redis.go`（CheckAndConsume/Refund 改 `Eval`）

- [ ] **Step 1: 移植 Lua 常量** — 把 Python `CHECK_AND_INCR`（多 tier 原子 check+INCRBY+EXPIRE，超则返回 `{tier_idx,ttl,current}`）+ `REFUND` 移植为 Go `const`（逐行忠实，KEYS/ARGV 顺序一致）。

- [ ] **Step 4: 测试** — Lua 原子性（并发 N goroutine CheckAndConsume，超限计数精确，无 race）；对照 Python lua_scripts 行为。

## 风险 / 注意

- **逐字段对齐**：以 Python `models.py` 为唯一基准；Go JSON tag 逐字段对照（含 `retry_after_seconds` snake_case）。
- **Lua 移植**：Python `CHECK_AND_INCR` 的 KEYS/ARGV 顺序 + 返回数组契约要忠实；Go `redis.Eval` 的 interface{} 返回解析（int64 / []interface{}）。
- **stash@{0} redis.go**：WIP 145 行参考但可能基于旧 key 格式（带 region）——R3a key 对齐 Python（无 region），WIP 的 key 部分要调整。
- **鉴权信任入口**：`INGRESS_SHARED_SECRET` env 各 overlay 一致；APISIX 路由 quota 时 key-auth 注入 X-Ingress-Auth（R1d 模式）。
- **Go 1.25**：构建环境 `go` 版本兼容（go.mod 1.25.0）。
- **Python quota 不删**：仅部署切 Go；Python 代码保留作 fallback/参考。
